## Pure-JS rewrite, smarter layout restore, and unit tests

### Fixed
- **Panel/auxiliary bar not restoring** — the extension now reads VS Code's internal
  workspace storage database (`state.vscdb`) at startup to seed the true initial panel
  visibility, rather than assuming all panels are open. Maximize → restore now correctly
  brings back only the panels that were actually visible before maximizing.

### Changed
- **Removed native SQLite dependency** — replaced `better-sqlite3` (which broke on every
  VS Code Node version update) with a pure-JS SQLite B-tree reader. No more
  `NODE_MODULE_VERSION` mismatches; the extension works across all VS Code versions
  without rebuilding native modules.

### Added
- Unit test suite for the SQLite reader — sweeps all local workspace databases to verify
  the reader never throws and always returns valid results.
- `VSCODE-API-REQUEST.md` — documents the upstream VS Code API proposal we filed so that
  panel visibility can eventually be tracked in real time
  ([microsoft/vscode#321409](https://github.com/microsoft/vscode/issues/321409)).

### Known limitation
VS Code does not expose a public API for reading panel visibility state mid-session.
The startup database probe is accurate at launch but won't reflect manual panel toggles
made after VS Code opens. See [microsoft/vscode#321409](https://github.com/microsoft/vscode/issues/321409)
— once that API ships, Max Editor will track state perfectly in real time.
